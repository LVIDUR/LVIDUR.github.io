文件处理命令

目录处理命令

  • mkdir [-p] [目录名] （-p 递归创建）

  • cd

    | cd | | |
    | —- | —- | ——————– |
    | cd | ~ | 进入当前用户的家目录 |
    | cd | - | 进入上次目录 |
    | cd | .. | 进入上一级目录 |
    | cd | . | 进入当前目录 |

  • pwd

  • ls

    | [选项] | [文件或目录] |
    | —— | ————————– |
    | -a | 显示所有文件，包括隐藏文件 |
    | -l | 显示详细信息 |
    | -d | 查看目录属性 |
    | -h | 人性化显示文件大小 |
    | -i | 显示inode |

  • rmdir [目录名]（只能删除空目录！）

  • rm -rf [目录名]

    • -r 删除目录
    • -f 强制（小心！Linux中文件删了就删了，没有回收站！ “rm -rf \”可将Linux中所有，所有东西删光，自杀hh）

  • cp [选项] [原文件] [目标目录]

    | 选项 | 作用 |
    | —- | ———————————- |
    | -r | 复制目录 |
    | -p | 连带文件属性复制（如修改时间） |
    | -d | 若源文件是链接文件，则复制链接属性 |
    | -a | 相当于 -pdr（上三个总和） |

  • mv [原文件] [目标目录] （剪切或改名）

链接命令

ln -s [原文件] [目标文件]

功能：生成链接文件

  • 硬链接
    • 拥有相同的i节点和储存block块，阔以看作是同一个文件
    • 可通过i节点识别
    • 不能跨分区
    • 不能针对目录使用
  • 软链接
    • 类似Windows快捷方式
    • 软链接拥有直接的i节点和block块，但是数据块中只保存原文件的文件名和i节点号，并没有实际的文件数据
    • 软链接的文件权限都为rwxrwxrwx
    • 修改任意文件，另一个都改变
    • 删除原文件，软链接不能使用

常用目录的作用

  • / 根目录
  • /bin 命令保存目录（普通用户就可以读取的命令）
  • /boot 启动目录，启动相关文件
  • /dev 设备文件保存目录
  • /etc 配置文件保存目录
  • /home 普通用户的家目录
  • /lib 系统库保存目录
  • /mnt 系统挂载目录
  • /media 挂载目录
